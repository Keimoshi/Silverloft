# Silverloft 产品需求文档 (PRD)

## 1. 产品概述
Silverloft 旨在打造一个轻量级、智能化的 SaaS 分析与报告生成平台。基于 Open Design 作为底层引擎，通过定制化的浅色 UI 主题、独立的后台管理机制与强大的大模型集成，提供简洁高效的用户体验与分析工具。

## 2. 核心目标
- **业务收敛与聚焦**：弱化基础账号等辅助功能，将核心功能重心放在数据分析、报告页面与核心内容的优化呈现。
- **环境隔离**：确保前端用户交互与后端模型接口调用完全解耦；普通用户端零可见敏感 API 配置，系统核心大模型相关配置参数严格收拢到后台由管理员进行控制。

## 3. 功能模块规划

### 3.1 前端门户体验优化 (进行中/已完成)
- **视觉体系换皮**：
  - 移除原有多余的页面及配置；
  - 提供清爽的 SaaS 浅色风格；
  - 强制全系统默认语言锁定为 `zh-CN`。
- **配置隐藏与引流**：
  - 在原设置面板中撤除所有的 API Key 与大语言模型直连配置。
  - 用户若需进行底层配置改动，统一提示跳转至 `/admin` 并受凭证管控。

### 3.2 独立后台权限系统 (已完成)
- **管理面板 `/admin`**：
  - 负责设置 API Providers（如 Anthropic, OpenAI 等）与系统 API 鉴权信息。
  - 提供唯一的管理入口并使用环境变量 `OD_ADMIN_TOKEN` 拦截所有未经授权的访问。
- **凭证控制**：密码固定且不对外明文展示，目前后台通行密码设定为 `[REDACTED]`。

### 3.3 核心分析功能扩展 (待规划重点)
- 围绕用户输入需求打造的各类“报告、信息图表或视图”。
- 完善数据对接链路，后续基于已隔离稳固的基础架构，专注强化前端分析页面的数据表达形式（看板视图、列表视图、独立图表生成等）。

## 4. 废弃或挂起功能
- 忘记密码 / 账号自助注册等全套基础登录认证体系。
- 复杂的模型参数下发暴露机制（已重构并移至后台）。
