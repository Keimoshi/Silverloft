# Silverloft 项目规划书

## 1. 项目背景与架构重组
由于初期 Vite + React 的纯前端架构方案在承接“真实数据流”与“模型配置安全”方面暴露出局限性，经过业务决策与技术讨论：
- **架构替换**：项目已整体接管并切换至 `open-design` 仓库作为全新底层平台 (`<repo-root>/open-design-port`)。
- **架构特点**：前后端完全分离与隔离，基于 Next.js（Web 端）与 Node.js（Daemon 服务端）运行。

## 2. 里程碑与阶段计划

### 第一阶段：基础基座搭建与安全隔离 (已完成)
- [x] 完成 `open-design` 的克隆并处理构建依赖与环境配置。
- [x] 确立本地化方案，强制配置前端语言为 `zh-CN`。
- [x] 构建轻量化浅色 SaaS 风格页面。
- [x] 剥离前端模型配置信息，建立隐藏的管理员通道 `/admin`，彻底杜绝了模型 API 及 Base URL 泄露的问题。
- [x] 完成并使用 `OD_ADMIN_TOKEN` 作为后台环境变量权限拦截控制。

### 第二阶段：核心分析功能强化与研发 (规划中 / Next)
- **目标**：在安全的 API 通道保护下，重点扩展系统针对用户意图的理解与反馈。
- **任务预估**：
  - 基于现有 UI 提供多样化的“看板图”、“数据图表”呈现形式。
  - 完成分析报告界面的业务对接，确保流畅的数据流输入与报告导出体验。
  - 后续研发团队全面投入，放弃一切低优附加功能的开发（如自助注册/找回密码等流程挂起）。

### 第三阶段：演示测试与灰度发布 (待排期)
- 规划内网环境或指定设备下的全量演示验收。
- 根据核心“分析功能”表现，决定未来的多模态生成方向（含现有音频、图片等特性的深入对接）。

## 3. 协作与交付准则
1. 涉及系统或接口环境的改动，必须确保不破坏原应用启动；严格执行 `pnpm typecheck` 以避免隐藏问题。
2. 凡是对外提供配置或关键凭据的代码，绝对不允许前端直接留存。
3. 日常开发采用单核调度，涉及实际代码变更时由前端/后端分配跟进，最终执行结果必须配有实际环境直连的证明截图进行验收。
